Element external Components Exchanges

Engineering Data

Major Kinds of Engineering Data > SOLUTION ELEMENT NEED description > Element external Components Exchanges

A (behavioral) component exchange is an interaction between two elements or components likely to exchange some items (information, signal, material, torque…). It realises functional exchanges between functions allocated to source and target components.

Activities producing the Data

  • Define & enforce a PBS and Component Integration Contract
  • Formalise Component development & integration contract
  • Perform a functional and non-functional Analysis

Activities using the Data

  • Perform Sub-System Engineering
  • Perform HW & mechanical engineering
  • Build IRS document (Interface Requirements Specification)
  • Build PIDS documents (Prime Item Development Specification) and subsystems, SW, HW specifications (SSS, SRS, HRS) including interfaces
  • Perform SW engineering

Usages

  • Solution elements interface description (ICD/IRS)
  • Software Requirements (SRS or equivalent in Agile)
