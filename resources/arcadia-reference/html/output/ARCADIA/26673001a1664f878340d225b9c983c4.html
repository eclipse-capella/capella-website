Operational constraints Identify operational non-functional Constraints Identify non-functional constraints

[non-functional] constraints describe quantitative and non-functional aspects of the actors and users behaviour and missions, such as latency requirements, safety issues, security constraints, need variability, and more.

They can be applied to any element of the need description, such as activities, interactions, operational processes, data...

A same consttraint can be applied to several elements.

Later in the process, each architecture definition decision will be confronted to each of these constraints, so as to ensure compliance of the system architecture with regards to need.
