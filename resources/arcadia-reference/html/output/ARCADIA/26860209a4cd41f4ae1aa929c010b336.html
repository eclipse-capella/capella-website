The Interface Trip

Engineering Process

Engineering Capabilities > Define a Solution satisfying Need > The Interface Trip

This process describes the way to define and justify internal and external interfaces, from need understanding to solution elaboration.

Interface management starts in system need analysis, for external systems and actors, down to physical architecture for internal interfaces. They are defined and justified thanks to functional analysis at each level, in collaboration between system and constituents engineering teams.

Interfaces are an important part of solution constituents specification: any modification should be validated in collaboration at physical architecture level.
