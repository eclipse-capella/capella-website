Traversing Non-functional Issues

Engineering Process

Engineering Capabilities > Define a Solution satisfying Need > Traversing Non-functional Issues

This process describes the way to take into account non-functional constraints (safety, security etc.) from need understanding to solution elaboration.

Non-functional specialties constraints (such as Safety, Security and more) are captured as part of need analysis and taken into account in solution architecture.

Each candidate architecture or architectural decision is challenged wrt each non-functional constraint. Iterations are performed until reaching an architectural compromise acceptable by all specialties and stakeholders.
