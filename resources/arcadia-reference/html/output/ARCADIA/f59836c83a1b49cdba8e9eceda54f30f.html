System Need Specification

Engineering Data

Major Kinds of Engineering Data > NEED Description > SYSTEM NEED > System Need Specification

The system need analysis and specification focuses on the system itself, in order to define how it can contribute to satisfy the former operational needs, along with its expected behavior and qualities.

Activities producing the Data

  • Perform an Operational Need Analysis
  • Perform Sub-System Engineering
  • Promulgate Reference Architecture
  • Perform Speciality or Discipline Analyses
  • Validate operational Need
  • Describe operational Context
  • Perform multi-Disciplines Trade-off
  • Link & evaluate customer Requirements

Activities using the Data

  • Validate conformance of each candidate architecture against design expectations incl. viewpoints
  • Drive a multi-parametric analysis (organisation, equipment, training, logistics...)
  • Define System Requirements
  • Define operational & Market Segmentation Variabilities
  • Define System required Capabilities
  • Describe operational Context
  • Define operational activities
  • Justify requirements fulfilment
  • Build OCD document (Operational Concept Description)
  • Choose the best trade-off between alternatives
  • Define customer deliveries based on operational capabilities
  • Validate conformance of each candidate architecture against design expectations incl. viewpoints
  • Select & validate best compromise architecture
  • Define functions to be achieved by system & users
  • Select and validate best compromise Architecture
  • Check & Ensure traceability/justification links with former perspective
  • Setup environment for each Project or Reference Configuration
  • Define IVVQ scenarios based on need analysis
  • Explore Solution Space & Alternatives
  • Formalise Component development & integration contract
  • Identify Missions / Goals of end users & stakeholders
  • Link & evaluate customer Requirements
  • Define system users/actors

Usages
