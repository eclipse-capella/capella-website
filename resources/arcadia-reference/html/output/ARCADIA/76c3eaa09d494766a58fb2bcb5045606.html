Implementation components

Engineering interaction

Engineering activities > Engineering Activities & Perspectives > DESIGN THE SOLUTION ARCHITECTURE > Design PHYSICAL ARCHITECTURE > Build candidate detailled Architectures > Implementation components

An implementation component (also known as a node) is a part of the final breakdown of the system (at the current engineering level), delivering resources for the behavioural components that it implements (heat, cooling, power, computation resources...).

Several components may be grouped, for complexity management and separation of concerns purpose, into one father component, so as to manage complexity and hide details inside the component. This leads to a component tree structuring the components(also known as component breakdown).

Source activity

  • Define implementation components to deliver processing resources

Destination activity

  • Check each engineering/architecting decision against physical viewpoints
