Spec. Functional Chains

Engineering Data

Major Kinds of Engineering Data > NEED Description > SYSTEM NEED > Spec. Functional Chains

A functional chain describes a usage of the system in a given context. It is a logical organization of functions and functional exchanges to fulfil a Capability.

Activities producing the Data

  • Define Functional Chains traversing the system
  • Validate functional Analysis
  • Define System required Capabilities
  • Perform a functional and non-functional Analysis
  • Check Requirements against need and early architecture

Activities using the Data

  • Define Product specification and commercial Offer variabilities
  • Finalise requirements based on logical architecture
  • Formalise and link system requirements with functional Analysis
  • Check & Ensure traceability/justification links with system need analysis
  • Validate each architecture against operational and functional/non functional need
  • Define Functional Chains traversing the system
  • Check & Ensure traceability/justification links with former perspective
  • Build SSS document (System/Subsystem Specification)
  • Setup environment for each Project or Reference Configuration
  • Validate functional Analysis
  • Define customer deliveries based on operational capabilities
  • Validate each architecture against operational and functional/non functional need
  • Justify requirements fulfilment
  • Explore Solution Space & Alternatives
  • Define IVVQ scenarios based on need analysis
  • Perform an internal functional Analysis
  • Define functional Scenarios
  • Allocate functions to the system and to users
  • Build OCD document (Operational Concept Description)
  • Check Requirements against need and early architecture
