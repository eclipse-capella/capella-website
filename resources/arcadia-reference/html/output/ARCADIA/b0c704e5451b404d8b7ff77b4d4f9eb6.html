Engineering Stakeholders Tasks



This is a description of tasks to be performed by some key Stakeholders contributing to systems engineering, as addressed by Arcadia.

Warning: the definition of stakeholders and of tasks is given only as an example, each engineering organisation having its own roles, activities and workshare. furthermore, this description is by no means exhaustive; it is just a sample of some stakeholders and tasks that may take benefit from Arcadia approach and outputs.

Note : this description may apply to other levels of (system, software, hardware…) engineering for the architecture definition and verification part, to some extent; it has to be adapted to each discipline and domain according to their own context.
