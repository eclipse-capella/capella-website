Notional Scenarios

Engineering Data

Major Kinds of Engineering Data > SOLUTION Description > LOGICAL ARCHITECTURE > Notional Scenarios

A design scenario describes the behavior of the system in a given context. It is a time-ordered set of functional exchanges between functions, or between the system and external actors, to fulfil a Capability.

Activities producing the Data

  • Finalise requirements based on logical architecture
  • Validate each architecture against operational and functional/non functional need
  • Explore Solution Space & Alternatives
  • Select best Compromise Architecture
  • Check each engineering/architecting decision against logical viewpoints
  • Select & validate best compromise architecture
  • Perform an internal functional Analysis

Activities using the Data

  • Validate conformance of each candidate architecture against design expectations incl. viewpoints
  • Check & Ensure traceability/justification links with logical architecture
  • Check against architecture drivers
  • Check each engineering/architecting decision against physical viewpoints
  • Finalise requirements based on logical architecture
  • Build SSDD document (System/Segment Design Document)
  • Check & Ensure traceability/justification links with logical architecture
  • Complement and detail internal functional Analysis
  • Perform Speciality or Discipline Analyses
  • Ensure consistency between Solution Architecture and product Variability
  • Finalise requirements based on physical architecture
  • Validate each architecture against operational and functional/non functional need
  • Check each engineering/architecting decision against logical viewpoints
  • Perform multi-Disciplines Trade-off
  • Select & validate best compromise architecture
  • Materialize logical components into behavioural Components breakdown
  • Define component interfaces and exchanges
  • Check & Ensure traceability/justification links with system need analysis
